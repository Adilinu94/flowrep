## 3. Was JETZT konkret offen ist, priorisiert

### 3.1 Adis eigentliche, noch unbeantwortete Frage: UX-Modell "Start"-Knopf
Adi hat das intendierte Modell so beschrieben: Nutzer wählt eine Übung,
erst nach Tippen auf "Start" beginnt die Zählung - nicht permanent. Bei
kurzer Prüfung in dieser Session gefunden: es gibt zwei separate,
möglicherweise nicht konsistente Wege in `home_screen.dart`:
- `_openCalibrationWizard` (Zeile ~292) → vermutlich der neue Guided-
  Calibration-2.0-Wizard.
- `_startRest()` → `_startCountdown()` → `_startRecording()` (Start-Knopf
  bei Zeile ~684) → ruft am Ende `widget.engine.startGuidedCalibration()`
  auf - das ist die ALTE, Ein-Wiederholung-Auto-Kalibrierung, nicht der
  neue Wizard-Pfad.

**Nicht mehr geklärt in dieser Session (Zeitdruck - Adi wollte den Chat
wechseln):** Wird `startGuidedCalibration()` hier bedingungslos aufgerufen,
auch wenn schon ein gültiges `ExerciseProfile` aus dem Wizard existiert
(`hasValidCalibration == true`)? Falls ja, ist das vermutlich Restcode aus
der Zeit vor dem Wizard und sollte durch einen Zweig ersetzt werden: bei
vorhandenem Profil direkt in `active`/Zählung übergehen, `startGuided-
Calibration()` nur noch als Fallback ohne Profil. **Das als ersten Schritt
in der neuen Session klären, bevor irgendwas daran geändert wird - im Code
nachsehen, nicht annehmen.**

### 3.2 Design-Frage, bewusst noch offen (keine mechanische Aktivierung mehr)
g_p ist jetzt live massgeblich, sobald es kalibriert ist - aber der dafür
verwendete Schwellenwert kommt weiterhin aus einer *eigenen, groben
Live-Bootstrap-Schätzung während der Session* (erste klare Bewegung, dann
wieder Ruhe → Schwelle = Peak × 0,5), NICHT aus dem sorgfältig per
Known-Count-Sweep im Wizard ermittelten `ExerciseProfile.theta`. Genauso
unverdrahtet: `chosenSignal`/`minRepIntervalSeconds`/`prominenceMin` aus
dem Profil.

Adis eigene Einschätzung dazu (wörtlich sinngemäß aus dem Chat): die
Live-Schätzung ist zum Testen ganz praktisch, weil sie sich jede Sitzung
neu anpasst - aber weniger fundiert als der Wizard-Wert. Echte
Produktentscheidung, keine Einzeiler-Aktivierung: wahrscheinlich sollte
der Wizard-Wert Vorrang haben, WENN ein gültiges Profil existiert, und die
Live-Schätzung nur der Fallback ohne Profil sein (ähnlich wie 3.1) - aber
das ist eine Vermutung dieser Session, keine getroffene Entscheidung.
Mit Adi absprechen, nicht selbst festlegen.
